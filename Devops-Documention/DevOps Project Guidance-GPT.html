Great question — this is the exact mental gap every DevOps beginner hits, and your confusion is 100% valid.

Answer (single clear paragraph):
Initially, your website runs on a local Node.js server only to validate the application and UI, not the DevOps tooling. At this stage, Grafana, dashboards, and monitoring are NOT tested—they don’t even exist yet. Once the UI works locally, you commit the code to GitHub (this completes the Source stage). The Build phase starts when Jenkins pulls that same GitHub code, builds a Docker image that includes your Node.js app, and runs it as a container; from this point onward, you stop caring about the local Node.js server. Monitoring tools like Grafana and Prometheus are connected only after the container is deployed on EC2, where real CPU, memory, and application metrics exist to monitor. In short: Node.js local server = app correctness, Docker container = build artifact, EC2 runtime = monitoring & dashboards—each tool activates only when its stage becomes relevant.

That’s the mental model professionals use.
